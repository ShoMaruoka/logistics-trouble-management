## ドキュメント管理

### 更新ルール
- 機能追加・変更時は必ずドキュメントを更新
- バージョン管理で変更履歴を追跡
- レビュー必須（技術的な内容）
- 定期的な見直し（四半期毎）

### ドキュメント形式
- **Markdown**: 基本形式
- **YAML**: API仕様書
- **JSON**: Postmanコレクション
- **PlantUML**: 図表（ER図、シーケンス図等）
